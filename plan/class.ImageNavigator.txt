jsiv ImageNavigator

The ImageNavigator class is responsible for acquiring images for the viewport
to display.  This class will handle providing the user with a FileDialog to
choose a file to open, navigating between the other files in the directory
that the user opened a file in, and reporting the file statistics for the
directory.

ImageNavigator will handle these with four public methods:
    openFile()      - Show FileDialog to let user pick an initial file
    refresh()       - Update the file list in case files have changed
    openNext()      - Load the next image in the current directory
    openPrevious()  - Load the previous image in the current directory

ImageNavigator will communicate through the ImageNavigatorListener interface.
When the ImageNavigator is created, it needs to be provided with an object
that implements the ImageNavigatorListener interface, and a JFrame to use as
the parent for the FileDialog and any alerts it needs to provide to the user
about problematic files.

The ImageNavigatorListener interface provides 2 methods:
    navigationAvailabilityChanged(boolean available)
    newImageLoaded(String imageName, BufferedImage image, int index, int total)

-------------------------------------------------------------------------------

openFile()

This method shows the user a FileDialog and allows them to pick a file to open.

If the user selects a file, then this method passes the directory and file
to the overloaded openFile(String path, String name).  Otherwise the user
cancelled the FileDialog, so it just returns and nothing further happens
with this request.

-------------------------------------------------------------------------------

openFile(String path, String name)

This overloaded openFile() method allows multiple paths to opening a file:
- openFile() /* no parameters */ will present the FileDialog for the user
            to pick a file to open
- ImageDropHandler will interpret the file being dropped onto the window
            and pass the path and filename so ImageNavigator can handle
            opening the file as it would for a file chosen another way

If the user chooses a file that is not an image, or is not an image that the
standard java implementation can handle (.png, .gif, .jpg, .jpeg, .bmp, .tif,
.tiff, .wbmp), or if the file fails to load for any reason, ImageNavigator
will show an Alert dialog advising the user that it was unable to load the
file they selected.  Then openFile() will abort without sending any signals.

If the user chooses a good file, but the file is moved or deleted before this
method can add it to a file list (a very minute chance) then ImageNavigator
will show an Alert dialog advising the user that the file was moved or
deleted, and then will abort without sending any signals.

If all goes well, openFile() will cache the loaded image file for future use,
and make a list of all of the image files in the directory that the user chose
a file in.  The files will be naievely matched based on their file extensions,
using the list above.  ImageNavigator will hold that list for future navigation.

Next, the method will emit navigationAvailabilityChanged() with true or false
depending on if there is more than 1 image file in the list, or if there is
only the one the user opened.

Finally, the method will emit newImageLoaded(), providing the image's filename,
the image itself, the index of the file within the list of files, and the total
number of files in the list.

-------------------------------------------------------------------------------

refresh()

This method refreshes the list of files in the current directory.  If the user
adds or removes files, they can use this to ensure the program works with the
updated contents of the directory.

If the currently cached image is gone, this refresh() will still emit the
newImageLoaded() with the cached image, but will list it as index 0 out of the
total number of files.  If there are no image files in the directory, then
it will also emit navigationAvailabilityChanged(false) to alert the other
components that there is no navigation available.

-------------------------------------------------------------------------------

openNext()
openPrevious()

These do the same thing, but in different directions.  The image file list
that ImageNavigator maintains will be sorted into alphabetical order.
openNext() moves forward (a -> z) through the list while openPrevious() moves
backward (z -> a) through the list.

When called, the ImageNavigator tries to open the next/previous filee in the
list.  If it is unable to open the file, it will remove that file from the
list so it does not try to open it again.  It will continue to do this until
it successfully opens a file, or it returns to the cached image.

If it removes all but the cached image, it will stop trying to load anything
else, emit navigationAvailabilityChanged(false) and emit
newImageLoaded(), providing the cached image and an index of 1, total of 1.

If it is able to load the next image in sequence, it will only emit
newImageLoaded() with the new image filename, new image data and the new
index/total.  ImageNavigator will cache this image for later use.

-------------------------------------------------------------------------------

navigationAvailabilityChanged()

This is a signal to other components that openNext()/openPrevious() are either
available or not available.  If they are available, openNext() and
openPrevious() will quickly load the next/previous image in the directory
where openFile() last opened an image.  If they are not available, the
ImageNavigator will ignore any openNext() and openPrevious() calls.

-------------------------------------------------------------------------------

newImageLoaded()

This is a signal to other components that a new image has been loaded and is
availablee to be displayed/worked on/whatever.  It will provide the image
filename, the image itself as a BufferedImage, a file index, and a total file
count.

The file count is a count of all of the image files in the directory where the
most recent file that was loaded is located.  Files are counted if they have
one of the above extensions (see openFile() above).

The index is the index into that list of files (sorted alphabetically).  So
if there are 3 files; a.png, b.png and c.png, and ImageNavigator just loaded
b.png, it will emit:

    "b.png"
    BufferedImage of b.png
    2
    3

In a rare edge case, it may emit a newImageLoaded with an index of 0.  This
represents that the cached image has been removed from the directory, but
the program is still displaying it because the user has not tried to open
or navigate to a different file.  Navigating to another file will fix the
index, but that removed file will no longer be accessible unless it is put
back into the directory or opened from a different location.
