jsiv ViewportListener

The ViewportListener is an interface that another class implements to respond
to the Viewport's requests and information.  Using this interface, the Viewport
does not need to know who the Listener is, and can assume the following
will be responded to in some manner:

    void requestOpenNext()
    void requestOpenPrevious()
    void imageSizeChanged(Dimension newImageSize)
    void viewportSizeChanged(Dimension newViewportSize)
    void zoomChanged(double newZoomLevel)
    void newColorUnderPointer(Color color)
    void requestCopyColorToClipboard(Color color)

--------------------------------------------------------------------------------

- void requestOpenNext()
- void requestOpenPrevious()

The Viewport will allow the user to click on a Previous Image button, or a
Next Image button if navigation is available.  When the user does this,
one of these will be called to alert the listener so they can take action.
The Viewport will make no assumptions, so if a new image is opened as a
result of this message, it will come in through a method call on the
Viewport.

--------------------------------------------------------------------------------

- imageSizeChanged(Dimension newImageSize)

When a new image is displayed, the Viewport will emit this message so
the listener can update with the new image size.  This size is the size
of the image at 100% zoom level.

--------------------------------------------------------------------------------

- viewportSizeChanged(Dimension newViewportSize)

When the Viewport is resized (as a result of the program's window being
resized), the Viewport will emit this message so the listener can update
with the new Viewport size.  This is only the size of the Viewport itself,
the JFrame that this Viewport is contained in is not considered.

--------------------------------------------------------------------------------

- zoomChanged(double newZoomLevel)

When the user zooms in or out, resets the zoom or opens a new image (which
will cause the Viewport to zoom in or out to size the image to the Viewport),
the Viewport will emit this message so the listener can update with the
new zoom level.

--------------------------------------------------------------------------------

- newColorUnderPointer(Color color)

When the mouse moves over the Viewport, the Viewport will emit this message
so the listener can update with the Color under the mouse pointer.  The
Viewport will provide the raw Color object, the listener is responsible
for decoding it or converting it as needed.

--------------------------------------------------------------------------------

- requestCopyColorToClipboard(Color color)

When the user right-clicks on a pixel in the Viewport, they are requesting
to copy that pixel color to the system clipboard so they can paste it into
another program.  When this happens, the Viewport will emit this message
with the raw Color object.  The listener is responsible for performing
any conversion as well as the actual copy-to-clipboard.
